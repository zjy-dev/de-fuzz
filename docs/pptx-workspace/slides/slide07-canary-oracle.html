3. Oracle 实例：Stack Canary 二分检测

跨 ISA 的 Canary 绕过检测预言机

核心原理

渐进增加缓冲区填充大小，观察程序退出状态的变化模式。利用 ret_addr、canary、buffer 三者位置关系的单调性进行二分搜索。

二分搜索算法

  • 在 [0, N] 区间二分查找最小崩溃 size
  • check(size): 运行程序，返回退出码
  • 若退出码=139 且有哨兵: 报告 Bug
  • 若退出码=134: Canary 正常工作
  • 哨兵机制排除函数内部间接崩溃

跨架构支持

使用 0x41 ('A') 填充构造非法地址，在 x64/AArch64/LoongArch64 上均触发 SIGSEGV。通过 QEMU 实现交叉架构执行。

哨兵输出机制

在 seed() 函数返回前打印 "SEED_RETURNED"，区分真正的返回地址被覆盖(哨兵已打印)与函数内部崩溃(哨兵未打印)。
