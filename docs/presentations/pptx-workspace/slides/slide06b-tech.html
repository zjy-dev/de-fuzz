3. 技术贡献 - GCC 插桩与静态分析

克服 GCC 静态分析的工程挑战

覆盖率插桩编译器构建

为 GCC 源码添加覆盖率插桩，支持多目标架构（x64/AArch64/LoongArch64）的交叉编译，实现对防御相关代码路径的精确追踪

CFG 静态提取

解析 GCC 中间表示，提取防御机制实现函数的控制流图（CFG），识别未覆盖基本块及其后继关系，指导目标选择

防御代码定位

自动识别编译器中与特定防御策略相关的源文件和函数，如 Stack Canary 的插入点、FORTIFY 的检查逻辑

覆盖率反馈闭环

将运行时覆盖率与静态 CFG 结合，动态更新 Line->Seed 映射，实现覆盖率驱动的模糊测试闭环

技术难点：GCC 代码库超过千万行，防御机制代码分散在多个模块，需精确定位并插桩目标代码路径
