1. 研究背景

编译器软件防御机制

现代编译器实现了多种软件防御机制，用于抵御内存安全漏洞攻击：

  • Stack Canary：检测栈缓冲区溢出
  • FORTIFY_SOURCE：强化危险函数的边界检查
  • SafeStack：隔离安全栈与不安全栈
  • CFI (Control Flow Integrity)：防止控制流劫持

问题：静默的逻辑漏洞

这些防御机制本身可能存在实现缺陷，导致防御静默失效：编译器不崩溃、不报错，但防护已失效。

传统编译器 Fuzzer 关注崩溃和编译错误，无法检测此类逻辑漏洞。

典型案例：CVE-2023-4039 - GCC AArch64 Stack Canary 绕过漏洞，影响数十亿设备
