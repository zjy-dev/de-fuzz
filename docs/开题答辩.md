### 1. 研究背景、研究现状

编译器软件防御机制相关的 fuzz 研究几乎为 0.
这涉及到 silent 的逻辑 bug, 编译器往往不会崩溃或报错, 但软件防御机制已经悄然失效. 传统的编译器 fuzzer 在此方面基本无效.

为此本课题借助 LLM 的代码理解能力, 生成/变异能触发防御机制的种子.

而 LLM 驱动的 Fuzzer 相关的研究近些年涌现了一些优秀的作品, 如 whitefox 和 hlpfuzz. whitefox 是白盒 fuzzer, 从源码中用 LLM 总结触发深层逻辑的 pattern 后基于它用 LLM 生成输入. HLPfuzz 是约束求解的思路, 选择出有价值的基本块用模型进行约束求解, 用渐进式的上下文构建解决了 LLM fuzzer 中上下文粒度的问题.

### 2. 研究目标、研究内容

研究目标首先要能找出现存的软件防御机制相关的 cve 如 [cve-2023-4039](https://rtx.meta.security/mitigation/2023/09/12/CVE-2023-4039.html).

研究内容要提到我们预言机的设计, 能找出逻辑 bug.

你来根据文档和经验规划内容, 并保证一定有我提到的内容.

### 3. 研究方案、初步实验

defuzz 目前已经可以找出软件防御机制失效相关的 cve 如[链接](https://rtx.meta.security/mitigation/2023/09/12/CVE-2023-4039.html). 

目前在 gcc 的 aarch64-v12.2, aarch64-v15.2(latest),  x64-v15.2(latest), loongarch64-v15.2上运行了实验.
